# pull과 pull request의 차이 
1. git pull (풀) 📥
"최신 버전을 내 컴퓨터로 다운로드하는 명령어"
무엇인가요? Git의 **명령어(Command)**입니다.
어디서 실행하나요? **내 컴퓨터(Local)**의 터미널에서 실행합니다. (git pull origin main)
어떤 일을 하나요? 원격 저장소(GitHub)의 최신 변경 사항을 내 컴퓨터로 다운로드하고, 현재 내 브랜치와 즉시 합칩니다(merge).
목적: 다른 팀원이 작업한 최신 코드를 내 컴퓨터에 반영하기 위해 사용합니다.
방향: 원격(GitHub) ➡️ 내 컴퓨터(Local)

2. Pull Request (풀 리퀘스트) 📤 - 팀에서 최종본을 합칠 때 merge 대신 사용 
"내 작업을 최종본에 합쳐달라고 깃허브에서 요청하는 기능"
무엇인가요? Git 명령어가 아니라, **GitHub(깃허브)의 핵심 협업 기능(Feature)**입니다.
어디서 실행하나요? 깃허브 웹사이트에서 버튼을 눌러 사용합니다.
어떤 일을 하나요?
-내가 작업한 브랜치(my-feature)를 push합니다.
-GitHub에서 "Pull Request 열기" 버튼을 누릅니다.
-팀 동료들이 내 코드를 검토(Code Review)하고 "승인(Approve)"합니다.
-"Merge" 버튼을 눌러 비로소 main 브랜치에 합쳐집니다.
